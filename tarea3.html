01 · Introducción

¿Qué es HTML5?

HTML significa HyperText Markup Language (Lenguaje de Marcado de Hipertexto). Es el lenguaje estándar para crear páginas web. No es un lenguaje de programación —no tiene lógica ni variables— sino un lenguaje de marcado: usas etiquetas para describir qué es cada parte del contenido.

HTML5 es la quinta y actual versión, lanzada oficialmente en octubre de 2014 por el W3C (World Wide Web Consortium). Introdujo decenas de nuevas etiquetas semánticas, soporte nativo para audio y video, el elemento <canvas> para gráficos, APIs de geolocalización, almacenamiento local y mucho más.

💡 Concepto clave

HTML define qué es el contenido (un título, un párrafo, una imagen). CSS define cómo se ve (color, tamaño, posición). JavaScript define qué hace (reaccionar a clics, animarse). Los tres juntos forman una página web completa.

02 · Estructura básica

La estructura básica de todo documento HTML5

Todo archivo HTML5 válido debe tener esta estructura mínima. Es el esqueleto sobre el que se construye cualquier página web:

index.html
<!-- 1. Declaración del tipo de documento -->
<!DOCTYPE html>
<!-- 2. Elemento raíz. lang indica el idioma -->
<html lang="es">

  <!-- 3. Sección de metadatos (no visible) -->
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <meta name="description" content="Mi primera página web"/>
    <title>Mi Primera Página</title>
    <link rel="stylesheet" href="styles.css"/>
  </head>

  <!-- 4. Sección de contenido visible -->
  <body>
    <h1>¡Hola, Mundo!</h1>
    <p>Esta es mi primera página web.</p>
    <script src="app.js"></script>
  </body>

</html>
<!DOCTYPE html>Declaración

Indica al navegador que el documento usa HTML5. Debe ser siempre la primera línea. No es una etiqueta HTML, es una instrucción para el navegador.

<html>Raíz

Es el elemento raíz que envuelve todo el documento. El atributo lang mejora la accesibilidad e indexación SEO.

langdirid
<head>Metadatos

Contiene información sobre la página que no se muestra al usuario: título, charset, CSS, JavaScript, metadatos para SEO y redes sociales.

<meta>Metadato · Vacía

Define metadatos del documento. charset establece la codificación de caracteres. viewport es esencial para diseño responsivo.

charsetnamecontenthttp-equiv
<title>Metadato

Define el título de la página que aparece en la pestaña del navegador y en los resultados de Google. Es obligatorio y muy importante para SEO.

<link>Metadato · Vacía

Vincula recursos externos al documento, principalmente hojas de estilos CSS. También se usa para favicons y fuentes de Google Fonts.

relhreftypemedia
<body>Contenido

Contiene todo el contenido visible de la página web: textos, imágenes, videos, formularios, tablas y cualquier otro elemento que el usuario pueda ver.

<script>Script

Vincula o contiene código JavaScript. Se recomienda colocarlo al final del body para que la página cargue más rápido.

srcdeferasynctype
03 · Sintaxis y reglas

Sintaxis y reglas fundamentales

HTML tiene una sintaxis muy simple. Conocer sus reglas te evitará errores comunes desde el principio.

¿Cómo funcionan las etiquetas?

La mayoría de las etiquetas tienen una etiqueta de apertura y una de cierre. El contenido va entre ambas:

sintaxis.html
<!-- Etiqueta con apertura y cierre -->
<etiqueta atributo="valor">Contenido</etiqueta>

<!-- Etiqueta vacía (no tiene contenido ni cierre) -->
<img src="foto.jpg" alt="Descripción"/>
<br/>
<hr/>
<input type="text"/>

<!-- Anidamiento correcto (de adentro hacia afuera) -->
<p>Texto con una <strong>palabra en negrita</strong> aquí.</p>

<!-- ❌ INCORRECTO: cruzar etiquetas -->
<p><strong>Texto</p></strong>

<!-- Los comentarios son así y no se muestran -->
<!-- Esto es un comentario -->

⚠️ Regla importante

HTML5 no distingue entre mayúsculas y minúsculas, pero la convención es siempre usar minúsculas en las etiquetas y atributos. <P> y <p> funcionan igual, pero se considera mala práctica usar mayúsculas.

04 · Elementos de bloque

Elementos de bloque vs. elementos en línea

Esta es una de las distinciones más importantes en HTML. Los elementos de bloque ocupan todo el ancho disponible y siempre empiezan en una nueva línea. Los elementos en línea solo ocupan el espacio que necesitan su contenido.

🟦 Bloque (Block)

<div> — contenedor
<p> — párrafo
<h1> — encabezado
<section> — sección
<ul> — lista
<table> — tabla

🩷 En línea (Inline)

<span> <a> <strong> <em> <img> <input> <label> <button>
05 · El contenedor universal

<div> y <span>: los contenedores genéricos

<div> es el contenedor de bloque genérico. No tiene significado semántico por sí solo, pero es utilísimo para agrupar elementos y aplicarles estilos CSS. <span> es su equivalente en línea.

contenedores.html
<!-- div agrupa elementos de bloque -->
<div class="tarjeta">
  <h2>Título de la tarjeta</h2>
  <p>Descripción del contenido.</p>
  <a href="#">Ver más</a>
</div>

<!-- span aplica estilos a texto en línea -->
<p>
  El precio es
  <span style="color: green;">$299</span>
  por mes.
</p>
06 · HTML Semántico

HTML Semántico: dale significado a tu código

El HTML semántico consiste en usar etiquetas que describen el significado de su contenido, no solo su apariencia. HTML5 introdujo muchas etiquetas semánticas nuevas que reemplazaron el uso excesivo de <div>.

Usar HTML semántico beneficia la accesibilidad (lectores de pantalla), el SEO (Google entiende mejor tu contenido) y la mantenibilidad del código.

Estructura semántica de una página HTML5

▬ <header> · cabecera
☰ <nav> · navegación
📄 <article>
📦 <section>
📦 <section>
◧ <aside> · barra lateral
— <footer> · pie de página
<header>Semántico

Cabecera de la página o de una sección. Generalmente contiene el logo, título principal y la navegación principal.

<nav>Semántico

Bloque de navegación principal. Contiene los enlaces de menú. Los lectores de pantalla saltan directamente a él.

<main>Semántico

Contenido principal y único de la página. Solo debe haber un <main> por página. Excluye encabezados, pies y barras laterales.

<article>Semántico

Contenido independiente y auto-contenido que podría funcionar por sí solo: una noticia, un post de blog, un comentario o una tarjeta de producto.

<section>Semántico

Sección genérica temática de un documento. Agrupa contenido relacionado. Generalmente tiene un encabezado propio <h2> o <h3>.

<aside>Semántico

Contenido relacionado pero secundario al principal. Barra lateral, anuncios, biografía del autor, notas complementarias.

<footer>Semántico

Pie de página o pie de sección. Contiene información de copyright, links legales, contacto, créditos y datos del autor.

<figure>Semántico

Figura auto-contenida: imagen, diagrama, código o ilustración con su <figcaption> (pie de figura) opcional.

07 · Encabezados

Encabezados: la jerarquía del contenido

Los encabezados van de <h1> a <h6>. Crean la jerarquía del documento, similar a los títulos y subtítulos de un libro. Solo debe haber un <h1> por página.

encabezados.html
<h1>Título principal de la página (solo uno)</h1>

  <h2>Sección principal</h2>
    <h3>Subsección</h3>
      <h4>Sub-subsección</h4>
        <h5>Nivel 5 (raro en práctica)</h5>
          <h6>Nivel 6 (muy raro)</h6>

  <h2>Otra sección principal</h2>
    <h3>Su subsección</h3>

✅ Buena práctica SEO

No uses los encabezados para dar tamaño al texto. Úsalos para organizar la jerarquía de información. El tamaño visual se controla con CSS. Google usa los encabezados para entender la estructura de tu contenido.

10 · Enlaces

Enlaces: la esencia del hipertexto

El elemento <a> (anchor = ancla) crea hipervínculos. Es la etiqueta que da vida al concepto de hipertexto que inventó Tim Berners-Lee. Puede enlazar a páginas, archivos, correos electrónicos o secciones de la misma página.

enlaces.html
<!-- Enlace externo -->
<a href="https://google.com" target="_blank" rel="noopener">
  Ir a Google
</a>

<!-- Enlace interno (otra página del mismo sitio) -->
<a href="contacto.html">Ir a Contacto</a>

<!-- Ancla (salta a un id en la misma página) -->
<a href="#seccion-contacto">Ir a Contacto</a>
<section id="seccion-contacto">...</section>

<!-- Enlace de correo -->
<a href="mailto:[email protected]">Enviar correo</a>

<!-- Enlace de teléfono -->
<a href="tel:+521234567890">Llamar</a>

<!-- Enlace para descargar archivo -->
<a href="doc.pdf" download>Descargar PDF</a>
AtributoDescripciónValores comunes
hrefURL de destino del enlace. Obligatorio.URL, #id, mailto:, tel:
targetDónde se abre el enlace._blank (nueva pestaña), _self (misma)
relRelación con el documento destino.noopener, noreferrer, nofollow
downloadDescarga el archivo en lugar de navegar.download o download="nombre.pdf"
titleTooltip que aparece al pasar el cursor.Cualquier texto

Imágenes, video y audio

  src="foto.jpg"

<video controls autoplay muted loop width="640">
  <source src="video.mp4"  type="video/mp4"/>
  <source src="video.webm" type="video/webm"/>
  Tu navegador no soporta video HTML5.

<!-- Audio HTML5 -->
<audio controls>
  <source src="cancion.mp3" type="audio/mpeg"/>
</audio>

<!-- Incrustar página o mapa -->
<iframe
  src="https://www.youtube.com/embed/ID"
  width="560" height="315"
  allowfullscreen

🔑 Atributo alt — accesibilidad

El atributo alt en las imágenes es obligatorio para la accesibilidad. Lo leen los lectores de pantalla para personas con discapacidad visual, y también lo usa Google para indexar imágenes en el buscador.

12 · Tablas

Tablas: datos en filas y columnas

Las tablas en HTML5 se usan para mostrar datos tabulares, no para hacer layouts de página (eso es trabajo de CSS Flexbox y Grid).

tabla.html
<table>
  <caption>Comparativa de lenguajes</caption>

  <!-- Encabezado de tabla -->
  <thead>
    <tr>
      <th scope="col">Lenguaje</th>
      <th scope="col">Tipo</th>
      <th scope="col">Nivel</th>
    </tr>
  </thead>

  <!-- Cuerpo de tabla -->
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Marcado</td>
      <td>Fácil</td>
    </tr>
    <tr>
      <td>Python</td>
      <td>Programación</td>
      <td colspan="2">Medio</td>
    </tr>
  </tbody>

  <!-- Pie de tabla -->
  <tfoot>
    <tr>
      <td colspan="3">Fuente: MDN Web Docs</td>
    </tr>
  </tfoot>
</table>
13 · Formularios

Formularios: recolectar información del usuario

Los formularios son el principal mecanismo de interacción del usuario con una web. HTML5 añadió docenas de nuevos tipos de <input> con validación nativa.

formulario.html
<form action="/enviar" method="POST">

  <!-- Texto simple -->
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre"
         placeholder="Tu nombre" required/>

  <!-- Email con validación automática -->
  <input type="email" name="email" required/>

  <!-- Número con rango -->
  <input type="number" min="0" max="100" step="5"/>

  <!-- Selector de fecha (HTML5) -->
  <input type="date" name="fecha"/>

  <!-- Selector de color (HTML5) -->
  <input type="color" value="#e8620a"/>

  <!-- Rango deslizable (HTML5) -->
  <input type="range" min="0" max="100"/>

  <!-- Casilla de verificación -->
  <input type="checkbox" id="acepto" name="acepto"/>
  <label for="acepto">Acepto los términos</label>

  <!-- Radio buttons -->
  <input type="radio" name="sexo" value="m"/> Masculino
  <input type="radio" name="sexo" value="f"/> Femenino

  <!-- Desplegable -->
  <select name="pais">
    <option value="mx">México</option>
    <option value="us">Estados Unidos</option>
  </select>

  <!-- Área de texto multilínea -->
  <textarea name="mensaje" rows="4" cols="50"></textarea>

  <!-- Botones -->
  <button type="submit">Enviar</button>
  <button type="reset">Limpiar</button>
  <button type="button">Acción JS</button>

</form>

Los atributos globales pueden usarse en cualquier etiqueta HTML. Son los más importantes de memorizar.

16 · Atributos data-*

Atributos data-*: guarda datos en el HTML

HTML5 permite crear atributos personalizados con el prefijo data-. Son muy útiles para almacenar información en elementos HTML que luego JavaScript puede leer sin hacer peticiones al servidor.

data-attrs.html
<!-- Guardamos datos en el HTML -->
<div
  class="producto"
  data-id="42"
  data-precio="299.99"
  data-categoria="electronica"
  data-en-stock="true"
>
  Smartphone X
</div>

<!-- JavaScript los lee así -->
<script>
const prod = document.querySelector('.producto');
console.log(prod.dataset.id);        // "42"
console.log(prod.dataset.precio);    // "299.99"
console.log(prod.dataset.enStock);   // "true"
</script>

18 · Buenas prácticas

Buenas prácticas al escribir HTML5

✅ SiempreEstructura

Declara siempre <!DOCTYPE html> al inicio, define lang en <html> y usa charset UTF-8 en <meta>.

✅ SemánticaCódigo limpio

Usa etiquetas semánticas (<header>, <nav>, <main>, <article>) en lugar de divs para todo. Tu código será más legible y accesible.

✅ Alt siempreAccesibilidad

Toda imagen <img> debe tener un atributo alt descriptivo. Si la imagen es decorativa, usa alt="" para que los lectores de pantalla la ignoren.

✅ Label-InputFormularios

Vincula cada <label> con su <input> usando el atributo for e id correspondiente. Mejora la usabilidad y accesibilidad.

⚠️ EvitaLayout

No uses tablas para estructurar el layout de la página. Las tablas son para datos tabulares. El layout es trabajo de CSS Flexbox y Grid.

⚠️ EvitaEstilos

No uses el atributo style="" en línea para estilar elementos. Escribe los estilos en un archivo CSS externo para mantener la separación de responsabilidades.

✅ IndentaciónLegibilidad

Usa 2 o 4 espacios de sangría para anidar etiquetas. El código bien indentado es más fácil de leer, depurar y mantener por otros desarrolladores.

✅ Un H1SEO

Usa solo un <h1> por página con la keyword principal. Los <h2> a <h6> pueden usarse múltiples veces siguiendo la jerarquía lógica.

🚀 Siguiente paso

Ya tienes los fundamentos de HTML5. El siguiente paso es aprender CSS3 para estilizar tus páginas, y después JavaScript para hacerlas interactivas. Con esas tres tecnologías puedes construir cualquier sitio web.
