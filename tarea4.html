01 · Introducción

¿Qué es CSS3?

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). Es el lenguaje que le da apariencia visual al HTML: colores, tipografías, tamaños, espaciado, animaciones y diseño de páginas completas.

CSS3 es la versión más reciente, lanzada en módulos desde el año 2011. Introdujo animaciones, transiciones, gradientes, variables, Flexbox, Grid y decenas de funcionalidades que antes solo eran posibles con JavaScript o imágenes externas.

💡 La trinidad de la Web

HTML es el esqueleto. CSS es la piel, ropa y estilo. JavaScript es el cerebro. Sin CSS, toda la web sería texto negro sobre fondo blanco, como un documento de texto plano.

02 · Sintaxis y reglas

Sintaxis: cómo se escribe CSS

CSS se escribe en reglas. Cada regla tiene un selector (a quién aplica) y un bloque de declaraciones (qué cambiar). Cada declaración es un par propiedad–valor separado por dos puntos y cerrado con punto y coma.

styles.css
/* selector  { propiedad: valor; } */

h1 {
  color: #e8620a;
  font-size: 2rem;
  font-weight: 700;
  text-align: center;
}

.tarjeta {
  background-color: white;
  border-radius: 12px;
  padding: 24px;
  box-shadow: 0 4px 20px rgba(0,0,0,0.1);
}

/* Múltiples selectores a la vez */
h1, h2, h3 {
  font-family: 'Playfair Display', serif;
}

/* Comentarios van entre /* y */
03 · Cómo agregar CSS

Tres formas de agregar CSS a tu HTML

index.html
<!-- 1. EXTERNO (recomendado) — archivo separado -->
<link rel="stylesheet" href="styles.css">

<!-- 2. INTERNO — dentro del <head> -->
<style>
  body { background: #f5f5f5; }
</style>

<!-- 3. EN LÍNEA — directamente en el elemento (no recomendado) -->
<p style="color: red; font-size: 18px;">Texto</p>

✅ Buena práctica

Siempre usa un archivo CSS externo. Mantiene el HTML limpio, permite reutilizar estilos en múltiples páginas y facilita el mantenimiento del proyecto.

04 · Selectores básicos

Selectores básicos

Los selectores determinan a qué elementos HTML se aplican los estilos. Son la parte más importante de CSS.

selectores.css
/* Selector de etiqueta — todos los <p> */
p { color: gray; }

/* Selector de clase — elementos con class="btn" */
.btn { background: blue; }

/* Selector de ID — el elemento con id="logo" */
#logo { width: 200px; }

/* Selector universal — todos los elementos */
* { box-sizing: border-box; }

/* Selector descendiente — <a> dentro de <nav> */
nav a { text-decoration: none; }

/* Selector hijo directo — solo hijos directos */
ul > li { list-style: none; }

/* Selector hermano adyacente */
h2 + p { margin-top: 0; }

/* Selector de atributo */
input[type="email"] { border-color: blue; }
a[href^="https"]    { color: green; } /* empieza con */
img[src$=".png"]    { opacity: 0.9; }  /* termina con */
05 · Selectores avanzados

Pseudoclases y pseudoelementos

Las pseudoclases seleccionan elementos en un estado especial. Los pseudoelementos crean partes virtuales de un elemento que no existen en el HTML.

pseudo.css
/* ── Pseudoclases de estado ── */
a:hover   { color: orange; }        /* al pasar el cursor */
a:active  { opacity: 0.7; }        /* al hacer clic */
a:visited { color: purple; }       /* enlace visitado */
input:focus { outline: 2px solid teal; }

/* ── Pseudoclases estructurales ── */
li:first-child    { font-weight: 700; }
li:last-child     { margin-bottom: 0; }
li:nth-child(2n)  { background: #f5f5f5; } /* pares */
li:nth-child(odd) { background: #fff; }    /* impares */
p:not(.especial)  { color: gray; }       /* negación */

/* ── Pseudoelementos (doble ::) ── */
p::first-line   { font-weight: 700; }    /* primera línea */
p::first-letter { font-size: 2em; }     /* primera letra */
::selection     { background: teal; }   /* texto seleccionado */

.btn::before {     /* inserta contenido antes */
  content: "→ ";
}
.titulo::after {   /* inserta contenido después */
  content: "";
  display: block;
  width: 40px; height: 3px;
  background: orange;
}
06 · Cascada y herencia

La Cascada: cómo CSS decide qué estilo gana

La C de CSS viene de Cascading (en cascada). Cuando múltiples reglas aplican al mismo elemento, CSS sigue un orden de prioridad para decidir cuál gana.

cascada.css
/* Orden de la cascada (de menor a mayor prioridad): */
/* 1. Estilos del navegador (User Agent)             */
/* 2. CSS externo / interno del autor                */
/* 3. CSS en línea (style="")                        */
/* 4. !important — fuerza el estilo sin importar     */

p { color: blue; }         /* aplicará... */
p { color: red; }          /* ...esto gana (último en declararse) */

/* !important sobreescribe todo (úsalo con cuidado) */
p { color: blue !important; } /* gana siempre */

/* ── Herencia ── */
/* Propiedades como color, font-family, line-height   */
/* se HEREDAN de padres a hijos automáticamente.      */
/* Propiedades como margin, padding, border NO se heredan. */

body {
  font-family: 'DM Sans', sans-serif; /* heredada por todos */
  color: #333;                        /* heredada por todos */
}

.no-heredar {
  color: inherit;    /* fuerza herencia */
  color: initial;   /* valor por defecto del navegador */
  color: unset;     /* hereda si es heredable, o initial */
}
07 · Especificidad

Especificidad: el sistema de puntos de CSS

Cuando dos reglas aplican al mismo elemento, CSS calcula la especificidad de cada selector. Gana el que tenga más puntos. Se calcula en tres columnas: (IDs, Clases, Etiquetas).

style="" (en línea)
1,0,0,0
#id
0,1,0,0
.clase / :pseudoclase / [attr]
0,0,1,0
etiqueta / ::pseudoelemento
0,0,0,1
* (universal)
0,0,0,0
especificidad.css
p                    { color: gray;   } /* (0,0,0,1) */
.texto               { color: blue;   } /* (0,0,1,0) ← gana */
#principal           { color: green;  } /* (0,1,0,0) ← gana más */
div#principal .texto p{ color: red; }   /* (0,1,1,2) ← mayor */
08 · Modelo de caja

El Modelo de Caja (Box Model)

Todo elemento HTML es una caja rectangular compuesta por 4 capas concéntricas. Entender el Box Model es fundamental para controlar el espaciado y tamaño de cualquier elemento.

MARGIN
BORDER
PADDING
CONTENT width × height
box-model.css
.caja {
  /* CONTENT — tamaño del contenido */
  width:  300px;
  height: 200px;

  /* PADDING — espacio interior (entre contenido y borde) */
  padding: 20px;           /* todos los lados */
  padding: 10px 20px;     /* arriba/abajo  izq/der */
  padding: 5px 10px 15px 20px; /* top der bot izq */

  /* BORDER — borde del elemento */
  border: 2px solid #333;
  border-radius: 12px;   /* esquinas redondeadas */

  /* MARGIN — espacio exterior (separa de otros elementos) */
  margin: 24px;
  margin: 0 auto;        /* centrar horizontalmente */
}

/* box-sizing: border-box es la mejor práctica */
/* hace que padding y border NO aumenten el tamaño total */
* {
  box-sizing: border-box; /* ← ponlo siempre */
}

10 · Colores

Colores en CSS3

CSS admite múltiples formatos para definir colores. CSS3 añadió HSL, RGBA, HSLA y la función oklch() moderna, que permiten controlar transparencia y luminosidad de forma intuitiva.

colores.css
/* Nombre de color */
color: tomato;
color: teal;

/* Hexadecimal */
color: #e8620a;       /* 6 dígitos */
color: #e83;          /* abreviado */
color: #e8620a80;    /* con transparencia (últimos 2 = alpha) */

/* RGB */
color: rgb(232, 98, 10);

/* RGBA — con canal de transparencia 0-1 */
color: rgba(232, 98, 10, 0.5);  /* 50% opacidad */

/* HSL — Hue (0-360°) Saturation% Lightness% */
color: hsl(24, 90%, 48%);

/* HSLA — con transparencia */
color: hsla(24, 90%, 48%, 0.8);

/* oklch — moderno, más uniforme visualmente */
color: oklch(0.65 0.20 35);

/* Gradientes */
background: linear-gradient(135deg, #e8620a, #c94a00);
background: radial-gradient(circle, #fff, #000);
background: conic-gradient(red, blue, green);

11 · Tipografía

Tipografía en CSS3

tipografia.css
/* Importar fuente de Google Fonts */
@import url('https://fonts.googleapis.com/css2?family=Playfair+Display:wght@700&display=swap');

p {
  font-family: 'DM Sans', sans-serif;  /* fuente + fallback */
  font-size: 1rem;          /* tamaño */
  font-weight: 400;         /* 100-900 */
  font-style: normal;      /* normal | italic | oblique */
  line-height: 1.6;        /* interlineado (sin unidad = relativo) */
  letter-spacing: 0.05em; /* espacio entre letras */
  word-spacing: 0.1em;   /* espacio entre palabras */
  text-align: left;        /* left | center | right | justify */
  text-decoration: none;  /* none | underline | line-through */
  text-transform: none;   /* none | uppercase | lowercase | capitalize */
  text-shadow: 2px 2px 4px rgba(0,0,0,0.3);
}

/* Fuente local personalizada */
@font-face {
  font-family: 'MiFuente';
  src: url('mifuente.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}

/* Tipografía fluida con clamp */
h1 {
  font-size: clamp(2rem, 5vw, 4rem);
}

14 · Display

La propiedad display

La propiedad display es la más importante para controlar el layout. Define cómo se comporta un elemento en el flujo del documento.

display.css
display: block;         /* ocupa toda la línea (div, p, h1) */
display: inline;        /* solo ocupa su contenido (span, a) */
display: inline-block;  /* inline pero acepta width/height */
display: none;          /* oculta el elemento (no ocupa espacio) */
display: flex;          /* activa Flexbox en el contenedor */
display: grid;          /* activa CSS Grid en el contenedor */
display: contents;      /* el elemento desaparece, sus hijos no */

/* visibility oculta pero SÍ ocupa espacio */
visibility: hidden;
visibility: visible;
15 · Posicionamiento

Tipos de posicionamiento

La propiedad position controla cómo se coloca un elemento en la página. Se combina con top, right, bottom, left y z-index.

posicionamiento.css
/* STATIC (por defecto) — flujo normal, ignora top/left */
position: static;

/* RELATIVE — se mueve relativo a su posición original */
position: relative;
top: 10px; left: 20px; /* el espacio original se conserva */

/* ABSOLUTE — se posiciona relativo al ancestro "positioned" */
position: absolute;
top: 0; right: 0;         /* esquina sup. derecha del padre */

/* FIXED — fijo respecto a la ventana, no se mueve al scrollear */
position: fixed;
top: 0; left: 0; right: 0; /* barra de navegación */

/* STICKY — relative hasta que llega a un umbral, luego fixed */
position: sticky;
top: 20px;               /* se "pega" a 20px del borde superior */

/* Z-INDEX — controla qué elemento queda encima */
z-index: 10;  /* mayor número = más al frente */
z-index: -1; /* detrás del contenido normal */
staticDefault

Flujo normal del documento. Las propiedades top/left/right/bottom no tienen efecto.

relativeRelativo

Se mueve relativo a su posición original. El espacio original se reserva. Crea contexto para absolute.

absoluteAbsoluto

Se posiciona relativo al ancestro position≠static más cercano. Sale del flujo normal.

fixedFijo

Fijo respecto a la ventana del navegador. No se mueve al hacer scroll. Usado en navbars.

stickyPegajoso

Comportamiento híbrido: relative hasta que el scroll llega al umbral, luego actúa como fixed.

z-indexProfundidad

Controla el eje Z (profundidad). Solo funciona en elementos con position ≠ static.

16 · Flexbox

Flexbox: el sistema de layout en una dimensión

Flexbox permite alinear y distribuir elementos en una fila o columna con control total. Es ideal para componentes: barras de navegación, tarjetas, botones centrados y layouts simples.

flexbox.css
/* ── Propiedades del CONTENEDOR (padre) ── */
.contenedor {
  display: flex;

  /* Dirección del eje principal */
  flex-direction: row;            /* → row | ← row-reverse */
  flex-direction: column;         /* ↓ column | ↑ column-reverse */

  /* Alineación en eje principal (horizontal si row) */
  justify-content: flex-start;    /* al inicio */
  justify-content: flex-end;      /* al final */
  justify-content: center;        /* al centro */
  justify-content: space-between; /* espacio entre items */
  justify-content: space-around;  /* espacio alrededor */
  justify-content: space-evenly;  /* espacio igual */

  /* Alineación en eje cruzado (vertical si row) */
  align-items: stretch;    /* estira a la altura del padre */
  align-items: flex-start; /* arriba */
  align-items: flex-end;   /* abajo */
  align-items: center;     /* centrado vertical */
  align-items: baseline;   /* por línea base del texto */

  /* Salto de línea */
  flex-wrap: nowrap;  /* no envuelve (default) */
  flex-wrap: wrap;    /* envuelve a la siguiente línea */

  /* Espacio entre items */
  gap: 16px;
  gap: 12px 24px; /* filas columnas */
}

/* ── Propiedades del ITEM (hijo) ── */
.item {
  flex-grow: 1;   /* crece para llenar espacio disponible */
  flex-shrink: 0; /* no se encoge aunque no haya espacio */
  flex-basis: 200px; /* tamaño base antes de crecer/encoger */
  flex: 1 1 200px;   /* shorthand: grow shrink basis */
  align-self: center; /* sobreescribe align-items solo para este */
  order: 2;          /* cambia el orden visual sin cambiar el HTML */
}
17 · CSS Grid

CSS Grid: layout en dos dimensiones

CSS Grid es el sistema de layout más poderoso de la web. Permite crear layouts en filas y columnas simultáneamente. Ideal para el layout completo de una página.

grid.css
/* ── Contenedor Grid ── */
.grid {
  display: grid;

  /* Definir columnas */
  grid-template-columns: 200px 1fr 1fr;   /* fija + 2 iguales */
  grid-template-columns: repeat(3, 1fr);   /* 3 columnas iguales */
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); /* responsivo */

  /* Definir filas */
  grid-template-rows: 80px 1fr auto;

  /* Espaciado */
  gap: 24px;
  column-gap: 24px; row-gap: 16px;

  /* Áreas nombradas */
  grid-template-areas:
    "header  header"
    "sidebar main  "
    "footer  footer";
}

/* Asignar áreas a los hijos */
.header  { grid-area: header;  }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main;    }
.footer  { grid-area: footer;  }

/* ── Item Grid — control de posición ── */
.item-grande {
  grid-column: 1 / 3;   /* ocupa de línea 1 a línea 3 (2 cols) */
  grid-row: 1 / 3;      /* ocupa 2 filas */
  grid-column: 1 / -1; /* hasta la última columna */
}
18 · Responsive / Media Queries

Diseño Responsivo y Media Queries

El diseño responsivo adapta la interfaz a diferentes tamaños de pantalla. Las Media Queries permiten aplicar estilos condicionales según las características del dispositivo.

responsive.css
/* Mobile First: estilos base para móvil, luego ampliar */

/* Estilos por defecto (móvil) */
.grid {
  grid-template-columns: 1fr;     /* 1 columna en móvil */
}

/* Tablet: pantallas ≥ 768px */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* Desktop: pantallas ≥ 1024px */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Desktop XL: pantallas ≥ 1280px */
@media (min-width: 1280px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}

/* Orientación */
@media (orientation: landscape) { .hero { height: 80vh; } }

/* Dark mode del sistema */
@media (prefers-color-scheme: dark) {
  body { background: #0b0f1a; color: #d4dce8; }
}

/* Reducir movimiento (accesibilidad) */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}
19 · Variables CSS

Variables CSS (Custom Properties)

Las variables CSS, introducidas en CSS3, permiten almacenar valores reutilizables. Son dinámicas: se pueden cambiar con JavaScript o dentro de Media Queries, a diferencia de las variables de SCSS.

variables.css
/* Declarar en :root para acceso global */
:root {
  --color-primary: #e8620a;
  --color-dark:    #0b0f1a;
  --font-body:     'DM Sans', sans-serif;
  --radius:        12px;
  --shadow:        0 4px 20px rgba(0,0,0,0.2);
  --spacing-sm:    8px;
  --spacing-md:    16px;
  --spacing-lg:    32px;
}

/* Usar con var() */
.btn {
  background:    var(--color-primary);
  border-radius: var(--radius);
  padding:       var(--spacing-sm) var(--spacing-md);
  box-shadow:    var(--shadow);
}

/* Valor de fallback si la variable no existe */
color: var(--color-accent, orange);

/* Cambiar con Media Query */
@media (prefers-color-scheme: dark) {
  :root { --color-dark: #ffffff; }
}

/* Cambiar con JavaScript */
// document.documentElement.style
//   .setProperty('--color-primary', '#ff0000');
20 · Transiciones

Transiciones CSS3

Las transiciones animan el cambio de estado de una propiedad (de un valor a otro). Son perfectas para efectos de hover, focus y cambios de clase con JavaScript.

transiciones.css
.btn {
  background: #e8620a;
  transform: scale(1);

  /* propiedad  duración  función  retraso */
  transition: background .3s ease 0s;

  /* Múltiples propiedades */
  transition:
    background  .3s ease,
    transform   .2s ease,
    box-shadow  .3s ease;

  /* O transicionar todo */
  transition: all .3s ease;
}

.btn:hover {
  background: #c94a00;
  transform: scale(1.05);
}

/* Funciones de timing */
/* ease       — lento → rápido → lento (default)  */
/* linear     — velocidad constante                */
/* ease-in    — empieza lento                      */
/* ease-out   — termina lento                      */
/* ease-in-out — lento al inicio y al final        */
/* cubic-bezier(x1,y1,x2,y2) — curva personalizada */
transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
21 · Animaciones

Animaciones con @keyframes

Las animaciones CSS permiten crear movimiento complejo sin JavaScript. Se definen con @keyframes y se aplican con la propiedad animation.

animaciones.css
/* 1. Definir la animación con @keyframes */
@keyframes aparecer {
  from  { opacity: 0; transform: translateY(20px); }
  to    { opacity: 1; transform: translateY(0); }
}

@keyframes pulsar {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.1); }
  100% { transform: scale(1); }
}

@keyframes girar {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

/* 2. Aplicar con animation */
.elemento {
  /* nombre  duración  timing  retraso  repeticiones  dirección  relleno */
  animation: aparecer .6s ease 0s 1 normal forwards;

  /* O por separado */
  animation-name:            aparecer;
  animation-duration:        .6s;
  animation-timing-function: ease;
  animation-delay:           .2s;
  animation-iteration-count: infinite; /* o un número */
  animation-direction:       alternate; /* va y vuelve */
  animation-fill-mode:       forwards;  /* mantiene el estado final */
  animation-play-state:      paused;    /* pausar con JS */
}

/* Animaciones escalonadas con delay */
.item:nth-child(1) { animation-delay: .1s; }
.item:nth-child(2) { animation-delay: .2s; }
.item:nth-child(3) { animation-delay: .3s; }
22 · Transformaciones

Transformaciones 2D y 3D

La propiedad transform mueve, rota, escala o sesga elementos sin afectar el flujo del documento. Es la base de la mayoría de las animaciones modernas.

transform.css
.elemento {
  /* ── Transformaciones 2D ── */
  transform: translateX(20px);         /* mover derecha */
  transform: translateY(-10px);        /* mover arriba */
  transform: translate(20px, -10px);   /* mover X e Y */

  transform: scale(1.2);              /* escalar 120% */
  transform: scaleX(0.5);             /* solo horizontal */

  transform: rotate(45deg);           /* rotar 45° */
  transform: rotate(-90deg);          /* rotar negativo */

  transform: skewX(15deg);            /* sesgar horizontal */
  transform: skewY(10deg);            /* sesgar vertical */

  /* Combinar múltiples transformaciones */
  transform: translateY(-8px) scale(1.05) rotate(2deg);

  /* ── Transformaciones 3D ── */
  transform: rotateX(30deg);
  transform: rotateY(45deg);
  transform: perspective(500px) rotateY(30deg);
  transform: translateZ(50px);

  /* Punto de origen de la transformación */
  transform-origin: center center;   /* default */
  transform-origin: top left;
  transform-origin: 0% 100%;
}

24 · Buenas prácticas

Buenas prácticas en CSS3

✅ ResetBase

Siempre incluye * { box-sizing: border-box; margin: 0; padding: 0; } al inicio para normalizar entre navegadores.

✅ VariablesMantenibilidad

Declara colores, tipografías y espaciados como variables en :root. Cambiar el tema completo será cuestión de editar una sola sección.

✅ Mobile FirstResponsive

Escribe los estilos base para móvil primero, luego usa min-width en media queries para pantallas más grandes. Es más eficiente.

✅ Clases semánticasNomenclatura

Nombra las clases según su propósito, no su apariencia. Usa .btn-primary no .btn-rojo. Si cambias el color rojo, el nombre sigue teniendo sentido.

⚠️ Evita !importantCascada

Usar !important rompe la cascada y crea bugs difíciles de resolver. Si necesitas usarlo, probablemente hay un problema de especificidad que corregir.

⚠️ Evita IDs en CSSEspecificidad

Los selectores de #id tienen especificidad muy alta, lo que dificulta sobreescribirlos. Usa clases para estilar y reserva los IDs para el HTML y JavaScript.

✅ Usa rem no pxAccesibilidad

Usa rem para tipografía y em o % para espaciados. Respeta la configuración de tamaño de fuente del usuario en su navegador.

✅ Organiza tu CSSEstructura

Organiza en secciones: reset → variables → base → layout → componentes → utilidades. Usa comentarios para separar cada bloque.

🚀 Siguiente paso

Ya dominas los fundamentos de CSS3. El siguiente paso natural es explorar SCSS/Sass para organización avanzada, o aprender Tailwind CSS para construir interfaces rápidamente con clases de utilidad. ¡Y después, JavaScript para darle vida completa a tus páginas!
